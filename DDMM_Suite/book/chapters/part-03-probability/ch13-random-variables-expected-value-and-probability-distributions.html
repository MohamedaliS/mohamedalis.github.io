13.1 Learning objectives

By the end of this chapter, the student should be able to:

  1. Define a random variable and distinguish discrete from continuous random variables.
  2. Construct and interpret a probability distribution for a discrete random variable.
  3. Calculate and interpret the expected value and variance of a discrete random variable.
  4. Apply expected value to simple business decisions.
  5. Explain what a probability distribution represents and how it differs from a frequency distribution.

13.2 Introduction

In previous chapters, data were treated as fixed: a column of observations that had already been collected. Probability shifts the focus to what has not yet happened. A random variable is a quantity whose value depends on the outcome of a random process. Before a sales representative makes their calls for the day, the number of deals they will close is unknown. After the calls, it is a fixed number. In the period before the calls, it is a random variable.

This chapter introduces random variables and their probability distributions, which describe the complete pattern of probabilities across all possible values. The expected value summarises the long-run average outcome and is the foundation of decision analysis under uncertainty.

The dataset for this unit contains performance records for 180 sales representatives, with variables covering deals closed, revenue, calls made, and training status.

13.3 Random variables

A random variable assigns a numerical value to each outcome in a sample space. It is conventionally denoted by a capital letter such as X.

A discrete random variable can take only distinct, countable values. The number of deals a sales representative closes in a month is discrete: it can be 0, 1, 2, 3, and so on, but not 1.7.

A continuous random variable can take any value within a range. Revenue generated is continuous: it can take any positive value, including fractional amounts.

13.4 Probability distributions for discrete variables

A probability distribution lists each possible value of a discrete random variable alongside its probability. Two conditions must hold:

  1. Each probability is between 0 and 1.
  2. The probabilities sum to 1.

13.5 Expected value

The expected value of a discrete random variable X, written \(E(X)\) or \(\mu\), is the probability-weighted average of all possible values:

\[E(X) = \sum_x x \cdot P(X = x)\]

It represents the long-run average outcome: if the random process were repeated many times, the average observed value would converge towards \(E(X)\).

Code 
expected_value <- sum(deals_dist$deals_closed * deals_dist$probability)
cat("Expected value E(X) =", round(expected_value, 3), "deals per month\n")
Expected value E(X) = 26.167 deals per month

An expected value of, say, 4.3 deals does not mean a representative will close 4.3 deals in any particular month. It means that over a large number of months, the average will approach 4.3.

13.6 Variance and standard deviation of a random variable

The variance of a discrete random variable measures the average squared deviation from the expected value:

\[\text{Var}(X) = \sigma^2 = \sum_x (x - \mu)^2 \cdot P(X = x)\]

The standard deviation is its square root, expressed in the original units.

Code 
mu <- expected_value
variance_X <- sum((deals_dist$deals_closed - mu)^2 * deals_dist$probability)
sd_X <- sqrt(variance_X)

var_results <- data.frame(
  Measure = c("Expected value E(X)", "Variance Var(X)", "Standard deviation SD(X)"),
  Value   = round(c(mu, variance_X, sd_X), 3)
)

knitr::kable(var_results,
  col.names = c("Measure", "Value"),
  caption   = "Table 13.2: Summary measures for the deals-closed distribution") |>
  kableExtra::kable_styling(bootstrap_options = c("striped"),
                             full_width = FALSE)
Table 13.2: Summary measures for the deals-closed distribution
Measure Value
Expected value E(X) 26.167
Variance Var(X) 153.791
Standard deviation SD(X) 12.401

A higher standard deviation indicates more variability in outcomes. Two sales teams might have the same expected deals closed per month but very different standard deviations, implying one is more consistent and the other more volatile.

13.7 Expected value in business decisions

Expected value is widely used to compare options when outcomes are uncertain.

Table 13.3: Expected value comparison for compensation options
Option Low outcome Mid outcome High outcome Expected value
Option A: Fixed contract GBP 3,000 (certain) GBP 3,000 (certain) GBP 3,000 (certain) GBP 3,000
Option B: Commission-based GBP 1,500 (P = 0.30) GBP 3,500 (P = 0.50) GBP 6,000 (P = 0.20) GBP 3,400

Option B has a higher expected value, but also higher variance. Whether Option B is preferable depends on the representative’s attitude to risk, not only on the expected value. A risk-averse individual may prefer the certainty of Option A despite the lower expected payout.

Note

Scenario. A training manager wants to know whether representatives who completed training have a higher expected number of deals closed. Use the dataset to compute and compare the expected values for each group.

Code 
ev_by_training <- df |>
  group_by(training_completed) |>
  summarise(
    n              = n(),
    mean_deals     = round(mean(deals_closed), 3),
    sd_deals       = round(sd(deals_closed), 3),
    mean_revenue   = round(mean(revenue_gbp), 0)
  )

knitr::kable(ev_by_training,
  col.names = c("Training", "n", "Mean deals (E[X])",
                "SD deals", "Mean revenue (GBP)"),
  caption   = "Table 13.4: Expected deals closed by training status",
  format.args = list(big.mark = ",")) |>
  kableExtra::kable_styling(bootstrap_options = c("striped", "hover"),
                             full_width = FALSE)
Table 13.4: Expected deals closed by training status
Training n Mean deals (E[X]) SD deals Mean revenue (GBP)
No 69 24.696 11.89 55,312
Yes 111 26.640 12.52 55,948

The mean in a sample is the empirical estimate of the expected value for that group. Comparing means across groups is therefore equivalent to comparing expected values from the underlying distributions.

13.9 Summary

A random variable assigns a numerical value to each outcome in a random process. Discrete random variables take countable values; continuous variables can take any value within a range. A probability distribution lists all possible values and their associated probabilities. The expected value is the probability-weighted average and represents the long-run mean. The variance measures the spread of the distribution around the expected value. Expected value is widely applied in business decision analysis, though it should always be considered alongside variance when outcomes vary substantially.

  2. A discrete random variable X can take values 0, 1, 2, 3 with probabilities 0.15, 0.35, 0.30, 0.20. Verify the distribution is valid, calculate E(X), and interpret the result.

  3. Using Table 13.4, which group has the higher expected deals closed? Is the difference in expected values sufficient to conclude that training is effective? What additional analysis would be required?

  4. Two investment options have expected returns of GBP 8,000 each. Option A has a standard deviation of GBP 500; Option B has a standard deviation of GBP 4,000. How would a risk-averse manager choose between them, and why?
