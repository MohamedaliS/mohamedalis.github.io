6.1 Learning objectives

By the end of this chapter, the student should be able to:

  1. Construct a frequency distribution for categorical and numerical data.
  2. Calculate relative frequencies and cumulative frequencies.
  3. Choose appropriate class intervals for a numerical frequency distribution.
  4. Interpret a histogram and a cumulative frequency curve.
  5. Use frequency distributions to answer practical questions about a dataset.

6.2 Introduction

Raw data is rarely informative on its own. A spreadsheet containing 120 insurance claim amounts tells a manager almost nothing until those numbers are organised. Frequency distributions provide that organisation. They group observations into categories or intervals and count how many fall into each group, converting an unreadable list into a structured summary.

This chapter introduces frequency distributions for both categorical and numerical data, extends them to relative and cumulative forms, and shows how histograms make the same information visual. The dataset for this unit contains records of 120 insurance claims, including claim amount, claim type, region, processing time, and settlement status.

6.3 What a frequency distribution is

A frequency distribution is a table that shows how many observations fall into each category or interval. The count in each class is the absolute frequency. Two derived measures add proportional context.

The relative frequency of a class is its absolute frequency divided by the total number of observations. It expresses each class as a proportion of the whole, and the relative frequencies across all classes sum to one.

The cumulative frequency of a class is the total number of observations in that class and all classes below it. Cumulative frequency answers questions of the form: how many observations lie below a given value?

6.4 Frequency distributions for categorical data

When the variable of interest is categorical, a frequency distribution simply counts the number of observations in each named group.

The insurance dataset contains a variable recording the type of each claim. The table below shows the frequency distribution for claim type.

Code 
library(dplyr)

cat_freq <- df |>
  count(policy_type, name = "Frequency") |>
  mutate(
    Relative_frequency = round(Frequency / sum(Frequency), 3),
    Percentage         = round(100 * Frequency / sum(Frequency), 1)
  ) |>
  arrange(desc(Frequency))

knitr::kable(
  cat_freq,
  col.names = c("Policy type", "Frequency", "Relative frequency", "Percentage"),
  caption   = "Table 6.1: Frequency distribution of insurance claims by policy type"
) |>
  kableExtra::kable_styling(
    bootstrap_options = c("striped", "hover"),
    full_width        = FALSE
  )
Table 6.1: Frequency distribution of insurance claims by policy type
Policy type Frequency Relative frequency Percentage
Health 35 0.292 29.2
Travel 30 0.250 25.0
Home 29 0.242 24.2
Motor 26 0.217 21.7

The relative frequency column shows what proportion of all claims each type accounts for. These proportions sum to one. The percentage column rescales them to a more familiar form.

6.5 Frequency distributions for numerical data

When the variable is numerical, observations must first be grouped into class intervals before counting. The choice of interval boundaries affects how the distribution looks and what it reveals.

6.5.1 Choosing class intervals

A practical rule is to use between 5 and 15 classes for most business datasets. Fewer than 5 classes obscures detail; more than 15 fragments the data into noise. A convenient starting formula is:

\[\text{Class width} \approx \frac{\text{Range}}{\text{Number of classes}}\]

Round the result to a convenient figure and keep all intervals equal in width. Open-ended classes at either extreme should be avoided where possible.

6.5.2 Building the table

The insurance dataset includes claim amounts ranging from small travel claims to large health and property claims. The table below groups these into intervals of GBP 2,000.

Code 
breaks <- c(0, 2000, 4000, 6000, 8000, 10000, 12000, 14000, 16000)
labels <- c("0-2,000", "2,001-4,000", "4,001-6,000", "6,001-8,000",
            "8,001-10,000", "10,001-12,000", "12,001-14,000", "14,001-16,000")

num_freq <- df |>
  mutate(class = cut(claim_amount_gbp, breaks = breaks,
                     labels = labels, include.lowest = TRUE)) |>
  count(class, name = "Frequency", .drop = FALSE) |>
  mutate(
    Relative  = round(Frequency / sum(Frequency), 3),
    Cumulative = cumsum(Frequency),
    Cum_rel    = round(cumsum(Relative), 3)
  )

knitr::kable(
  num_freq,
  col.names = c("Claim amount (GBP)", "Frequency", "Relative frequency",
                "Cumulative frequency", "Cumulative relative frequency"),
  caption   = "Table 6.2: Frequency distribution of claim amounts"
) |>
  kableExtra::kable_styling(
    bootstrap_options = c("striped", "hover"),
    full_width        = FALSE
  )

Claim amount (GBP) Frequency Relative frequency Cumulative frequency Cumulative relative frequency

2,001-4,000 73 0.608 108 0.900
4,001-6,000 12 0.100 120 1.000
6,001-8,000 0 0.000 120 1.000

10,001-12,000 0 0.000 120 1.000

6.6 The histogram

A histogram is the graphical form of a numerical frequency distribution. Bars are drawn for each class interval, with their height representing the frequency count. Unlike a bar chart, the bars in a histogram are adjacent because the underlying variable is continuous.

Code 
ggplot(df, aes(x = claim_amount_gbp)) +
  geom_histogram(breaks = breaks, fill = "#1B3A5C",
                 colour = "white", alpha = 0.85) +
  scale_x_continuous(labels = scales::comma) +
  labs(x = "Claim amount (GBP)", y = "Frequency",
       title = "Distribution of insurance claim amounts by GBP 2,000 intervals") +
  theme_dmm()

The histogram shows that the distribution is right-skewed: most claims are modest in value, but a small number of large claims extend the right tail. This pattern is common in insurance data and has practical implications for setting reserves and pricing.

6.6.1 The effect of bin width

Changing the number of bins changes the appearance of the distribution without changing the underlying data. The figure below shows the same claim amounts plotted with narrow bins (GBP 500) and wide bins (GBP 4,000).

Narrow bins reveal more detail but can create a jagged appearance that looks noisy. Wide bins smooth the shape but may hide features of the data. The appropriate bin width depends on the sample size and the purpose of the display.

6.7 The cumulative frequency curve

Plotting cumulative frequency against the upper boundary of each class produces the cumulative frequency curve, sometimes called an ogive. The curve rises from zero to the total number of observations and takes on an S-shape when the underlying distribution is approximately symmetric.

Code 
cum_plot_df <- data.frame(
  upper = breaks[-1],
  cum   = cumsum(table(cut(df$claim_amount_gbp, breaks = breaks, include.lowest = TRUE)))
)

ggplot(cum_plot_df, aes(x = upper, y = cum)) +
  geom_line(colour = "#1B3A5C", linewidth = 1.1) +
  geom_point(colour = "#C0392B", size = 2.5) +
  scale_x_continuous(labels = scales::comma) +
  labs(x = "Claim amount (GBP)", y = "Cumulative frequency",
       title = "Cumulative frequency of insurance claims") +
  theme_dmm()

The curve can be used to read off approximate percentiles. The value on the x-axis corresponding to a cumulative frequency of 60, for example, is the approximate 50th percentile for this sample of 120 claims.

6.8 Worked example

Note

Question. An operations manager wants to know what proportion of claims in this dataset were settled within 20 processing days. Use the frequency distribution to answer this question.

Code 
processing_breaks <- c(0, 5, 10, 15, 20, 25, 30, 35, 40, 45, 50)

proc_freq <- df |>
  mutate(class = cut(processing_days, breaks = processing_breaks,
                     include.lowest = TRUE)) |>
  count(class, name = "Frequency", .drop = FALSE) |>
  mutate(
    Cumulative     = cumsum(Frequency),
    Cumulative_pct = round(100 * Cumulative / sum(Frequency), 1)
  )

knitr::kable(

  caption   = "Table 6.3: Frequency distribution of processing days"
) |>
  kableExtra::kable_styling(
    bootstrap_options = c("striped", "hover"),
    full_width        = FALSE
  )

6.9 Summary

A frequency distribution organises raw data into classes and counts the observations in each. Absolute frequencies count cases; relative frequencies express proportions; cumulative frequencies accumulate from the lowest class upward.

For categorical data, the classes are the named groups of the variable. For numerical data, the analyst must choose class intervals, balancing detail against clarity. A histogram displays a numerical frequency distribution graphically, with bar height representing frequency and adjacent bars reflecting the continuous nature of the variable. The cumulative frequency curve plots the running total against class boundaries and allows approximate percentile reading.

6.10 Key terms

Table 6.4: Key terms for Chapter 06
Term Definition
Frequency distribution A table showing the number of observations falling into each category or interval.
Absolute frequency The count of observations in a given class.
Relative frequency The proportion of observations in a given class, calculated as the class frequency divided by the total.
Cumulative frequency The total number of observations in a given class and all classes below it.
Class interval A range of values used to group observations in a numerical frequency distribution.
Histogram A bar chart for numerical data in which bars represent class intervals and bar height represents frequency.
Ogive A graph of cumulative frequency plotted against the upper boundary of each class interval.
