12.1 Learning objectives

By the end of this chapter, the student should be able to:

  1. Apply the general addition rule to events that are not mutually exclusive.
  2. Apply the multiplication rule for independent and dependent events.
  3. Distinguish permutations from combinations and calculate each.
  4. Use counting rules to calculate probabilities in business contexts.
  5. Construct and use a probability tree for multi-stage processes.

12.2 Introduction

Chapter 11 introduced simple probability. Most real decisions involve sequences of events or selections from groups where the rules for combining probabilities matter. Should two defective items appearing in the same batch cause alarm? How many ways can a committee be formed from a pool of candidates? What is the probability that all three items in a quality sample pass inspection?

This chapter extends the probability toolkit with the general addition rule, the multiplication rule, and counting methods. The dataset for this unit contains 250 quality control batches from a manufacturing plant, with defect counts, shift information, and supervisor presence data.

12.3 The general addition rule

Chapter 11 showed that for mutually exclusive events, \(P(A \text{ or } B) = P(A) + P(B)\). When events are not mutually exclusive, simply adding their probabilities double-counts outcomes where both occur. The general addition rule corrects for this:

\[P(A \cup B) = P(A) + P(B) - P(A \cap B)\]

12.3.1 Example from the quality data

Code 
library(dplyr)

n_total <- nrow(df)

# Event A: batch is from the Night shift
# Event B: supervisor was not present
p_night   <- mean(df$shift == "Night")
p_no_sup  <- mean(df$supervisor_present == "No")
p_both    <- mean(df$shift == "Night" & df$supervisor_present == "No")
p_either  <- p_night + p_no_sup - p_both

results <- data.frame(
  Event   = c("P(Night shift)", "P(No supervisor)",
              "P(Night AND no supervisor)",
              "P(Night OR no supervisor)"),
  Value   = round(c(p_night, p_no_sup, p_both, p_either), 3)
)

knitr::kable(results,
  col.names = c("Event", "Probability"),
  caption   = "Table 12.1: General addition rule applied to quality data") |>
  kableExtra::kable_styling(bootstrap_options = c("striped", "hover"),
                             full_width = FALSE)
Table 12.1: General addition rule applied to quality data
Event Probability
P(Night shift) 0.396
P(No supervisor) 0.444
P(Night AND no supervisor) 0.192
P(Night OR no supervisor) 0.648

Without the correction term, adding the first two probabilities would overstate the combined probability by double-counting batches that were both on the night shift and had no supervisor present.

12.4 The multiplication rule

The probability that two events both occur is:

\[P(A \cap B) = P(A) \times P(B \mid A)\]

For independent events, \(P(B \mid A) = P(B)\), so:

\[P(A \cap B) = P(A) \times P(B)\]

12.4.1 Multi-stage quality inspection

Code 
# Defect rate overall
p_defect_per_item <- sum(df$defects_found) / sum(df$items_inspected)

# P(all 3 items pass) if inspections are independent
p_pass    <- 1 - p_defect_per_item
p_all_3   <- p_pass^3
p_at_least_1_defect <- 1 - p_all_3

mult_results <- data.frame(
  Quantity = c("P(item defective)", "P(item passes)",
               "P(all 3 pass | independent)",
               "P(at least 1 defective)"),
  Value = round(c(p_defect_per_item, p_pass, p_all_3,
                  p_at_least_1_defect), 4)
)

knitr::kable(mult_results,
  col.names = c("Quantity", "Probability"),
  caption   = "Table 12.2: Multiplication rule for independent inspections") |>
  kableExtra::kable_styling(bootstrap_options = c("striped"),
                             full_width = FALSE)
Table 12.2: Multiplication rule for independent inspections
Quantity Probability
P(item defective) 0.0538
P(item passes) 0.9462
P(all 3 pass &#124; independent) 0.8472
P(at least 1 defective) 0.1528

The complement rule is applied in the final row. Rather than calculating the probability of exactly one defect, exactly two, and exactly three separately and adding them, it is simpler to subtract the probability that none are defective from 1.

12.6 Counting methods

12.6.1 The fundamental counting principle

If a process has k stages and stage i has \(n_i\) possible outcomes, the total number of outcomes across all stages is:

\[n_1 \times n_2 \times \cdots \times n_k\]

A production process with 4 product lines, 3 shifts, and 8 machines has \(4 \times 3 \times 8 = 96\) possible combinations of these factors.

12.6.2 Permutations

A permutation is an ordered arrangement of objects. The number of ways to arrange r objects chosen from n distinct objects is:

\[P(n, r) = \frac{n!}{(n-r)!}\]

Order matters in a permutation. The number of ways to assign a first, second, and third prize to 3 of 10 finalists is \(P(10, 3) = 10 \times 9 \times 8 = 720\).

12.6.3 Combinations

A combination is a selection of objects where order does not matter. The number of ways to choose r objects from n distinct objects is:

\[\binom{n}{r} = \frac{n!}{r!(n-r)!}\]

The number of ways to select a 3-person audit team from 10 candidates is \(\binom{10}{3} = \frac{10!}{3! \cdot 7!} = 120\).

Code 
# Quality audit: how many ways to select 4 machines from 8 for inspection?
n <- 8; r <- 4
combinations <- choose(n, r)
permutations  <- factorial(n) / factorial(n - r)

counting_results <- data.frame(
  Method      = c("Combinations (order irrelevant)",
                  "Permutations (order matters)"),
  Formula     = c("C(8,4) = 8! / (4! x 4!)",
                  "P(8,4) = 8! / (8-4)!"),
  Result      = c(combinations, permutations)
)

knitr::kable(counting_results,
  col.names = c("Method", "Formula", "Result"),
  caption   = "Table 12.4: Counting selected machines for audit") |>
  kableExtra::kable_styling(bootstrap_options = c("striped"),
                             full_width = FALSE)
Table 12.4: Counting selected machines for audit
Method Formula Result
Combinations (order irrelevant) C(8,4) = 8! / (4! x 4!) 70
Permutations (order matters) P(8,4) = 8! / (8-4)! 1680

12.7 Worked example

Note

Scenario. A quality manager selects 2 batches at random from the 10 most recent night-shift batches for detailed review. What is the probability that both selected batches had no supervisor present, if 4 of the 10 were unsupervised?

Code 
# Without replacement: dependent events
p_first_unsupervised  <- 4/10
p_second_unsupervised <- 3/9   # one unsupervised batch already selected

p_both_unsupervised <- p_first_unsupervised * p_second_unsupervised

# Using combinations: C(4,2) / C(10,2)
p_combinations <- choose(4, 2) / choose(10, 2)

worked_results <- data.frame(
  Method = c("Multiplication rule (dependent)",
             "Combinations approach"),
  Calculation = c("(4/10) x (3/9)",
                  "C(4,2) / C(10,2)"),
  Probability = round(c(p_both_unsupervised, p_combinations), 4)
)

knitr::kable(worked_results,
  col.names = c("Method", "Calculation", "Probability"),
  caption   = "Table 12.5: Two equivalent approaches to the same problem") |>
  kableExtra::kable_styling(bootstrap_options = c("striped"),
                             full_width = FALSE)
Table 12.5: Two equivalent approaches to the same problem
Method Calculation Probability
Multiplication rule (dependent) (4/10) x (3/9) 0.1333
Combinations approach C(4,2) / C(10,2) 0.1333

Both methods give the same result. The multiplication rule is more intuitive for small problems; the combinations approach generalises more easily to larger selections.

12.8 Summary

The general addition rule, \(P(A \cup B) = P(A) + P(B) - P(A \cap B)\), applies when events can overlap. The multiplication rule gives the probability that two events both occur: \(P(A \cap B) = P(A) \times P(B \mid A)\), which simplifies to \(P(A) \times P(B)\) for independent events. Probability trees organise multi-stage calculations systematically.

Counting methods determine how many outcomes are possible. Permutations count ordered arrangements; combinations count unordered selections. Both are used to calculate probabilities when outcomes are equally likely.

## The multiplication rule

The probability that two events both occur is:

$$P(A \cap B) = P(A) \times P(B \mid A)$$

For **independent** events, $P(B \mid A) = P(B)$, so:

$$P(A \cap B) = P(A) \times P(B)$$

## Probability trees

A probability tree organises multi-stage probability calculations visually. Each branch represents one possible outcome at a stage, labelled with its probability. The probability of any complete path is the product of probabilities along that path.

```{r}
#| label: ch12-tree-table
#| echo: false
tree <- data.frame(
  Path = c("Night shift → No supervisor → Defect",
           "Night shift → No supervisor → No defect",
           "Night shift → Supervisor → Defect",
           "Night shift → Supervisor → No defect",
           "Other shift → No supervisor → Defect",
           "Other shift → No supervisor → No defect",
           "Other shift → Supervisor → Defect",
           "Other shift → Supervisor → No defect"),

    p_night * p_no_sup * (p_defect_per_item * 1.4),
    p_night * p_no_sup * (1 - p_defect_per_item * 1.4),
    p_night * (1-p_no_sup) * p_defect_per_item,
```

## Counting methods

### The fundamental counting principle

If a process has k stages and stage i has $n_i$ possible outcomes, the total number of outcomes across all stages is:

$$n_1 \times n_2 \times \cdots \times n_k$$

A production process with 4 product lines, 3 shifts, and 8 machines has $4 \times 3 \times 8 = 96$ possible combinations of these factors.

### Permutations

A **permutation** is an ordered arrangement of objects. The number of ways to arrange r objects chosen from n distinct objects is:

$$P(n, r) = \frac{n!}{(n-r)!}$$

Order matters in a permutation. The number of ways to assign a first, second, and third prize to 3 of 10 finalists is $P(10, 3) = 10 \times 9 \times 8 = 720$.

### Combinations

A **combination** is a selection of objects where order does not matter. The number of ways to choose r objects from n distinct objects is:

$$\binom{n}{r} = \frac{n!}{r!(n-r)!}$$

The number of ways to select a 3-person audit team from 10 candidates is $\binom{10}{3} = \frac{10!}{3! \cdot 7!} = 120$.

